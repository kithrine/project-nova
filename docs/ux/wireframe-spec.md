# Wireframe Specification

## Required screen states

Every major screen must define:

- Loading
- Empty
- Error
- Permission denied
- Restricted
- Success
- Disabled
- Blocked
- Concurrent update

## Responsive behavior

- Desktop: persistent navigation and dense workspaces
- Tablet: collapsible navigation
- Mobile: single column and sticky primary actions

## Critical components

- Journey Timeline
- Lifecycle Timeline
- Next Step Card
- Task List
- Blocker List
- Status Transition Control
- Application Workspace
- Placement Workspace
- Timesheet Review Card
- Incident Form
