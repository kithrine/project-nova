# Wireframe Specification

## Required screen states

Every major screen must define:

- Loading
- Empty
- Error
- Permission denied
- Restricted
- Success
- Disabled
- Blocked
- Concurrent update

## Responsive behavior

Design mobile-first: start from the smallest viewport and progressively enhance. The Operations experience is desktop-optimized for information density but is still built from the mobile-first baseline.

- Mobile: single column and sticky primary actions; navigation in the native details/summary disclosure
- Tablet and up (`md+`): the persistent full-height teal sidebar (brand pass 2026-07-16)
- Desktop: dense workspaces beside the sidebar

## Critical components

- Journey Timeline
- Lifecycle Timeline
- Next Step Card
- Task List
- Blocker List
- Status Transition Control
- Application Workspace
- Placement Workspace
- Timesheet Review Card
- Incident Form
