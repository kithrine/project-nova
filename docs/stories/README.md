# Stories

Stories are grouped by epic. Each story is normally built on its own branch and merged via one pull request (`docs/architecture/git-workflow.md`). Each implementation story should include:

- User value
- Scope
- Acceptance criteria
- Authorization requirements
- Lifecycle impacts
- Data impacts
- Accessibility requirements
- Test requirements
- Out-of-scope notes

## Status values

Each story's Status is one of: **Draft** (not yet fully specified), **Ready for Development** (fully specified and buildable), **Blocked — pending policy validation** (specified, but a legal/grant/policy question must be resolved first — see `docs/planning/open-questions.md`), **In Progress**, or **Done**.
