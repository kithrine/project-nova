# Product Requirements Document

## Product name

Project Nova

## Product summary

Project Nova is a workflow-driven workforce case-management platform that coordinates grant-funded transitional employment, beginning with placements at animal shelters.

## Goals

- Run one pilot cohort from application through employment outcome.
- Reduce coordinator administrative burden.
- Give applicants clear visibility into next steps.
- Give shelters a focused supervision workspace.
- Preserve an auditable record of decisions, hours, and outcomes.
- Protect sensitive participant information.

## Non-goals for MVP

- Public job marketplace
- Full payroll processing
- Blended funding
- AI matching
- Full chat platform
- Mobile native apps
- Dark mode
- Generic workflow builder
- Advanced analytics suite

## Primary personas

- Applicant / Participant
- Program Coordinator
- Shelter Supervisor
- Shelter Manager
- Grant Administrator
- Nova Administrator

## MVP capabilities

### Participant
- Learn how the program works
- Apply
- Save draft
- Upload required document metadata/files
- View application progress
- Complete onboarding
- View placement
- Record work hours
- View certifications

### Operations
- Review applications
- Conduct eligibility review
- Record interviews
- Record background decisions
- Accept or reject applications
- Manage enrollment and onboarding
- Track training and certifications
- Match participants to shelters
- Create and manage placements
- Track placement lifecycle
- Add case notes
- Manage incidents
- Generate basic reports

### Shelter
- Review proposed placements
- View assigned participants
- Approve timesheets
- Submit evaluations
- Report incidents

## Success measures

- Applications completed
- Time from submission to decision
- Participants accepted
- Participants reaching ready-for-matching
- Placements activated
- Hours worked and approved
- Placement completion
- Credentials earned
- Permanent employment outcomes
- Shelter participation and retention

## Key constraints

- Sensitive background data must be restricted.
- Application, enrollment, match, and placement are separate lifecycle objects.
- Lifecycle transitions are action-based, not unrestricted status edits.
- Server-side authorization is mandatory.
