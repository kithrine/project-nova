# Styling Philosophy

## Approved tools

- Tailwind CSS
- DaisyUI
- Native CSS
- CSS Modules
- CSS variables
- Keyframes
- Container queries
- Modern selectors

## Rule

Tailwind is the default, not a limitation.

Use native CSS when it improves:

- Readability
- Maintainability
- Animation quality
- Complex layout
- Visual identity
- Performance

## DaisyUI boundary

Use DaisyUI for common primitives where useful. Project Nova owns the final component behavior and appearance.

## Avoid

- Giant unreadable class strings
- Styling logic mixed with business rules
- Rigid Tailwind-only decisions
- Inconsistent one-off values
